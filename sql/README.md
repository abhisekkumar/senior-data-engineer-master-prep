# SQL interview practice

These employer-neutral examples consolidate the nine private legacy SQL scratch files. They are organized into window functions, analytics, and data-quality patterns. SQL Server date syntax is used where dialect-specific functions are necessary; each query uses fictional table and column names and contains no production data.

Every `.sql` file in this directory is automatically registered in the local dashboard when it
opens or when `make sync` runs. Add new solutions under the closest topic folder. Optional
`-- Title:`, `-- Difficulty:`, and `-- Primary Pattern:` comments control the displayed metadata;
otherwise the tracker safely derives a title and uses medium difficulty without altering the SQL.
