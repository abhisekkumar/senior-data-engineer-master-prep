# Agent guide

- Inspect before editing and preserve useful existing work, notes, tests, and progress.
- Never invent LeetCode numbers; use `unknown_` plus a TODO when uncertain.
- Follow `docs/QUESTION_TEMPLATE.py`. Include meaningful clarifying questions, examples, brute-force and optimal approaches, time/space complexity, recognition clues, edge cases, dry runs, and interviewer follow-ups.
- Prefer simple, interview-readable solutions over clever or obscure code.
- Add or update tests and tracker metadata with every question change.
- Avoid unnecessary dependencies, abstractions, and over-engineering.
- Run tests and validation before finishing; summarize changed files and unresolved issues.
- Never commit secrets or API keys. Update docs when commands or structure change.
- Treat `Senior Data Engineer Interview Prep/` as private local source material. Never stage, publish, or quote it directly. Migrate only generalized content after removing employer names, personal details, real identifiers, internal architecture, and confidential interview material.
- When migrating the author's solution, preserve its implementation exactly. Add educational sections around it; do not replace, optimize, reformat, or silently correct the code. Record syntax/correctness issues for author review.
